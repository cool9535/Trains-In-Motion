Contributing

    • Models must match the Mod's Art-style.
    • "units" may be whatever measurement form you please, models can be scaled up/down to match.
    • Wheel base must be 12 units (the rail width).
    • Chassis must be at least 20 units (so it can fit the player).
    • Length and height must be to scale with the chassis width.
    • Must be a .java model. Supported editors are:
      • SMP Toolbox 2 (preferred). NOTE: Shapes aren't supported, instead use a Shapebox part type on boxes.
      • FMC Modeler
      • Techne (No link is available currently)
      • MCModeler (No link is available currently, also untested)
    • Parts intended to animate such as wheels and pistons must be centered (for example dimension of 1 would be an offset of -0.5)
    • If the model is not from SMP Toolbox, it must be at least texturemapped, however it is preferred that they be fully textured (see textures)
    • Models must not infringe any forms of copyright (that means no chuggington, Thomas the tank engine, polar express, etc)
    • I will not accept .obj models, yes the render supports them but due to performance issues, I won't use them
    • Textures should be provided in 2 resolutions so they can properly support players of different GPU strengths
      • 16 times the base resolution (so for example if the model resolution is 512x512, the texture should be 8192x8192).
      • 4 times the base resolution (so for example if the model resolution is 512x512, the texture should be 2048x2048).
    • Textures must be shaded, the render does not do this for us, and if that feature is ever added it will only be an after-effect.
    • Textures must be built off base colors with known intention that they can and will have hues swapped later on.
    • If the textures are compressed, tell me ahead of time how you did it so I know whether or not to use my own forms of compression.
  • NOTE: the rules for this section are intended only as guidelines, I don't take any responsibility for your add-ons so I don't really care what you do.
    • Models should follow the size conventions mentioned before (I assume you want them to fit on the rails, and for the player to fit in the thing)
    • Models should not use shapes unless you make your own render to fix the shape rendering. So you probably still wanna use shapeboxes.
    • You can use the obj render if you want, just remember the entire model has to be cached on load and held on to throuought gameplay. So remember that when someone complains about RAM use.
    • Textures should follow the texture guidelines, if you wanna do different resolutions that's cool but the render treats your stuff the same as mine unless you make your own render.
    • While the resources are already there to allow you to make your own add-on mod to Trains in Motion, I will be making a tutorial for making addons, after the release of 0.2.7
